# Version2 UI/UX 改進方案 v2.1

## 📋 執行概述

本文檔基於**漸進式揭露 (Progressive Disclosure)** 原則，為 version2 設計針對「資料科學家」與「產業專家」的平衡化 UI。

---

## 🎯 核心設計原則

### 1. 漸進式揭露 (Progressive Disclosure)

由於 AutoML 涉及高度複雜的參數與流程，UI 應採用分層設計：

#### 對產業專家（預設視角）
```
簡化視圖：
┌─────────────────────────────────────┐
│ 📤 一鍵上傳資料                      │
│ ✓ 自動清洗 ✓ 自動特徵工程          │
│ 🚀 開始訓練                          │
└─────────────────────────────────────┘
        ↓
┌─────────────────────────────────────┐
│ 🏆 排行榜：XGBoost 94.7%            │
│ 💡 推薦行動 → 客戶流失風險高        │
│ 📊 What-If 模擬器 → 調整卡優惠      │
└─────────────────────────────────────┘
```

#### 對資料科學家（進階視角）
```
專業視圖：
┌────────────────────────────────────────┐
│ [進階設定] ← 點擊展開                 │
│                                        │
│ ├─ XGBoost 超參數調整                │
│ │  ├─ max_depth: [3-15]             │
│ │  ├─ learning_rate: [0.01-0.5]     │
│ │  └─ subsample: [0.5-1.0]          │
│                                        │
│ ├─ 交叉驗證設定                      │
│ │  └─ n_splits: 5 (Stratified K-Fold)│
│                                        │
│ └─ 自訂損失函數                       │
│    └─ Custom Loss.py: [上傳檔案]    │
└────────────────────────────────────────┘
```

**實現方式：** 
- 使用 `<details>` + `<summary>` HTML 元素
- CSS 平滑展開動畫
- 狀態持久化 (localStorage)
