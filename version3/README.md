# 🤖 AutoML 端對端平台 - UI/UX 設計方案

## 📋 項目介紹

此項目根據**「漸進式揭露」與「狀態可視化」**的核心設計原則，構建了一個專業級的 AutoML（自動化機器學習）平台。該平台兼顧技術深度與商業決策需求，為商業專家和資料科學家提供雙軌使用者體驗。

---

## 🎯 核心設計原則

### 1. 雙軌使用者體驗 (Dual-Track UX)
- **商業專家**：預設展示高階指標、簡單圖表總結、What-If 假設性模擬器
- **資料科學家**：提供展開/進階設定按鈕，查看數據清洗、特徵工程細節、超參數空間與日誌

### 2. 狀態與進度極度透明 (Hyper-Transparent Status)
- 全局進度條展示：數據清洗 → 特徵工程 → 模型尋優 → 評估
- 預估剩餘時間與即時日誌滾動視窗
- 每個步驟的完成狀態清晰可見

### 3. 將預測轉化為行動 (Action-Oriented)
- 儀表板不僅展示數字，更強調其意義
- 透過 UI 導引使用者進行決策
- What-If 模擬器支持即時參數調整與預測結果更新

## 📄 關鍵頁面詳解

### 1️⃣ 儀表板 (Dashboard)

**快速統計卡片：**
- 活躍實驗數量
- 已部署模型數量
- 平均準確率
- 平均訓練時間

**進行中的實驗進度：**
```
進度條設計：
[✓ 已完成] [✓ 已完成] [● 進行中] [待進行]
```
- 視覺化 Pipeline 階段
- 預估剩餘時間

**最近活動日誌：**
- 成功通知（✓ 綠色）
- 一般通知（ℹ️ 藍色）
- 警告通知（⚠️ 橙色）

---

### 2️⃣ 數據集管理 (Datasets)

**數據健康度評分 (Data Health Score)：**
- 雷達圖或評分環：直觀顯示缺失值、異常值、數據不平衡
- 健康度百分值（100% 為最佳）

**自動化處理提示：**
```
「已自動填補 5% 的缺失值」
「已將日期拆解為年、月、星期特徵」
```
- 建立對系統的信任感
- 展示 AutoML 的智能性

---

### 3️⃣ 實驗室 (Experiments)

**無程式碼配置表單：**
- 步驟 1：選擇目標變數
- 步驟 2：自動偵測任務類型（分類或回歸）
- 步驟 3：時間序列模式選項

**Pipeline 視覺化：**
- Sankey 圖顯示數據流
- 節點代表各個處理階段
- 連線粗細代表數據量

---

### 4️⃣ 模型排行榜 (Leaderboard)

**可排序的資料表：**
| 排名 | 演算法 | F1-Score | 準確率 | 訓練時間 | 操作 |
|------|--------|---------|--------|---------|------|
| 🥇 | XGBoost | 0.94 | 92.1% | 2h 15m | [詳情] [部署] |
| 🥈 | LightGBM | 0.91 | 89.5% | 45m ⚡ | [詳情] [部署] |

**徽章標示：**
- [🏆 最佳表現] - 指標得分最高
- [⚡ 最快推論] - 推論延遲最低

**一鍵比較工具：**
- 勾選 2-3 個模型
- 並排比較混淆矩陣、殘差圖、ROC 曲線

---

### 5️⃣ 洞察與決策 (Insights & Decisions) 🌟

**系統核心亮點**

#### A. 全域特徵重要性 (Global Feature Importance)
- 水平長條圖
- 顯示哪些特徵對預測結果影響最大
- 支持交互式點擊查詢

#### B. SHAP Value 瀑布圖 (Waterfall Plot)
- 針對單一筆預測
- 視覺化各特徵的正向/負向影響
- 綠色（正向）vs 紅色（負向）

#### C. What-If 模擬器（互動式拉桿）💡
**設計邏輯：**
```
┌─────────────────────────────────┐
│ 左側：控制面板                  │ 右側：實時結果
├─────────────────────────────────┤
│ 行銷預算: [■■■■□] 50%         │ 📊 預測購買機率
│ 交貨時間: [■■□□□] 0 天         │ 78.5%
│ 客戶滿意度: [■■■□□] 5.0        │
│ 產品價格: [■■■■] $100          │ 💡 建議：增加
│                                  │ 行銷預算以提高
│ 🔄 即時更新                      │ 購買機率。
└─────────────────────────────────┘
```

**用例：**
- 業務人員拖曳拉桿模擬不同場景
- 實時重新計算預測結果與勝率
- 直接將模型轉化為決策工具

## 📊 核心功能演示

### What-If 模擬器示例

當使用者調整「行銷預算」滑塊：
```javascript
// 預測邏輯簡化示例
預測機率 = 基線(50%) + 
           行銷預算影響(+25%) -
           交貨時間影響(-10%) +
           客戶滿意度影響(+20%) -
           價格影響(-15%)
           = 78.5%
```

建議系統會根據預測結果動態生成建議：
```
✓ 78.5% 購買機率 → 「優異的購買機率！建議立即執行」
◆ 50-75% 範圍 → 「可考慮微調行銷策略」
✗ <50% → 「建議增加行銷預算」等
```

## 🎓 使用情景

### 場景 1：商業經理
```
目標：評估增加行銷預算對銷售的影響
流程：
1. 進入「洞察與決策」頁面
2. 調整「行銷預算」滑塊到 80%
3. 觀看預測購買機率從 60% 提升至 85%
4. 根據決策建議作出投資決定
```

### 場景 2：資料科學家
```
目標：分析特定客戶的預測理由
流程：
1. 進入「洞察與決策」頁面
2. 查看 SHAP Value 瀑布圖
3. 確認「購買頻率」對預測的正向影響
4. 確認「年齡因素」的負向影響（如有）
5. 調整特徵工程策略
```

### 場景 3：模型選型
```
目標：在多個演算法間選擇最優模型
流程：
1. 進入「模型排行榜」頁面
2. 查看各演算法的準確率、訓練時間、推論延遲
3. 點擊「🏆 最佳表現」徽章的 XGBoost 模型
4. 點擊「部署」或「一鍵比較」進行冠軍比較
5. 決定最終部署方案
```

## 🎯 UX 最佳實踐應用

此項目體現了以下 UX 最佳實踐：

1. **信息層次化**
   - 默認顯示 20% 的信息（摘要）
   - 進階用戶可展開 80% 的詳細信息

2. **漸進式揭露 (Progressive Disclosure)**
   - 商業用戶看到簡化版面
   - 資料科學家看到完整技術細節

3. **明確的狀態反饋**
   - 進度條、加載動畫、即時日誌
   - 使用者永不迷茫

4. **行動導向的設計**
   - 每個指標都可轉化為決策
   - What-If 模擬器將「假設」轉為「行動」

5. **色彩心理學**
   - 綠色（成功、正向）
   - 紅色/橙色（警告、負向）
   - 藍色（信任、穩定）
