# 🚀 AutoML 平台 - 快速啟動指南

## 📌 項目概述

此項目是根據 **UI/UX 設計指南** 實現的完整 AutoML 端對端平台網站。

- **技術棧**：純前端（HTML5 + CSS3 + JavaScript）
- **圖表庫**：ECharts 5.4.2
- **無框架依賴**：輕量級、開箱即用
- **完全響應式**：桌面/平板/手機適配

## 🎮 核心交互功能演示

### 1. 儀表板 (Dashboard)

**入口：** 左側導覽 → 📊 儀表板

**主要功能：**
- 4 個快速統計卡片
- 2 個進行中的實驗進度
- 3 項最近活動日誌

**試玩提示：** 滑動頁面查看完整內容

### 2. 數據集管理 (Datasets)

**入口：** 左側導覽 → 📂 數據集管理

**main 特色：**
- 3 個數據集卡片示例
- 圓形健康度評分規（SVG）
- 自動化處理提示

**互動：** 點擊卡片可查看詳細信息（可擴展）

### 3. 實驗室 (Experiments)

**入口：** 左側導覽 → 🧪 實驗室

**主要功能：**
- 無程式碼 3 步驟配置表單
- ECharts Sankey Pipeline 視覺化

**試玩提示：**
1. 選擇數據集
2. 選擇目標變數
3. 點擊「開始訓練」
4. 觀看 Sankey 圖顯示數據流

### 4. 模型排行榜 (Leaderboard)

**入口：** 左側導覽 → 🏆 模型排行榜

**主要功能：**
- 5 列表格（演算法、指標、時間等）
- 🏆 最佳表現徽章
- ⚡ 最快推論標記
- 一鍵比較按鈕

**試玩提示：** 
- XGBoost 被標記為最高分
- LightGBM 被標記為最快

### 5. 洞察與決策 (Insights) 🌟 **核心亮點**

**入口：** 左側導覽 → 💡 洞察與決策

#### A. 全域特徵重要性
- **圖表類型**：水平長條圖
- **內容**：10 個特徵的影響力排行
- **交互**：Hover 查看數值

#### B. SHAP Value 瀑布圖
- **圖表類型**：瀑布圖
- **內容**：8 個階段顯示特徵的正向/負向影響
- **色彩**：綠色(+) vs 紅色(-)

#### C. What-If 模擬器 💡 **必試功能**

**操作指南：**

```
左側面板                    右側結果面板
─────────────────────────────────────────
行銷預算:    [■■■■□] 50%   📊 預測購買機率
交貨時間:    [■■□□□]  0 天  78.5%
客戶滿意度:  [■■■□□]  5.0  
產品價格:    [■■■■]  $100   💡 決策建議
```

**試玩步驟：**
1. 拖曳「行銷預算」滑塊到 80%
   → 預測機率提升至 ~85%
   → 建議：「優異的購買機率！建議立即執行」

2. 拖曳「交貨時間」滑塊到 +3 天
   → 預測機率降低至 ~65%
   → 建議：「建議縮短交貨時間以提高購買機率」

3. 調整「客戶滿意度」到最高 (10.0)
   → 預測機率提升至 ~92%

**預測邏輯（簡化）：**
```
基線值 50% +
行銷預算影響 (±25%) -
交貨延遲影響 (±10%) +
客戶滿意度影響 (±20%) -
價格敏感度 (±15%) =
最終預測結果
```

### 6. 部署與 API (Deployments)

**入點：** 左側導覽 → 🚀 部署與 API

## 🐛 常見問題 (FAQ)

### Q1：為什麼某些互動沒有反應？
**A：** 部分功能（如「開始訓練」）是演示模式，會彈出消息框。真正互動的是 What-If 模擬器、視圖切換、圖表縮放。

### Q2：圖表為什麼不顯示？
**A：** 檢查 ECharts CDN 是否加載。打開 F12 → 控制檯查看是否有錯誤。確保 `div` ID 匹配 (`feature-importance-chart` 等)。

### Q3：手機上某些元素重疊？
**A：** 這是響應式設計的知道行為。在手機上應自動調整為單欄佈局。刷新頁面或清除緩存重試。

### Q4：數據是真實的嗎？
**A：** 否，全部為示例數據。生產環境需連接真實數據源（API、數據庫等）。

### Q5：可以修改顏色和佈局嗎？
**A：** 完全可以！所有顏色在 `style.css` 的 CSS 變數中修改；佈局在 Grid/Flexbox 屬性中修改。
